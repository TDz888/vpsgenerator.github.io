Quick Start Guide
  1. Get GitHub Token with repo + workflow permissions
  2. Get Tailscale Auth Key (enable Reusable)
  3. Enter username/password or use random generator
  4. Click "Create VM" and wait for completion (3-5 minutes)
  5. Access VM via noVNC or RDP using Tailscale IP
Frequently Asked Questions
How long does VM creation take?
Usually 3-5 minutes. You can monitor progress in GitHub Actions tab of the created repository.
How long does the VM run?
Each VM runs for 6 hours maximum, then automatically shuts down. The workflow will keep it alive for exactly 6 hours.
Why is my VM stuck in "creating" status?
Check GitHub Actions logs. Common issues: token expired, missing permissions, GitHub service disruption, or Tailscale key invalid.
How to connect to the VM?
Use noVNC web console (http://[Tailscale-IP]:6080/vnc.html) or RDP client with Tailscale IP address. Username/password as set during creation.
What permissions does the GitHub token need?
The token requires repo (full control) and workflow permissions to create repository and trigger GitHub Actions.
